Strengths

In the Giant blockchain we have combined and upgraded the best practices previously developed by three separate blockchain teams. The idea of smart contracts comes from Ethereum, the masternode governance is borrowed from Dash, and the closest equivalent of our current Proof-of-Stake consensus model can be found in PIVX. By combining the best aspects of each of these technologies, Giant has the following advantages:

PoS & MN

Proof-of-Stake (PoS) and masternodes (MN) – while the former decreases the GIC volatility by making its price independent from miners (and making GIC eco-friendly), the latter is a reliable governance technology. Masternodes form a decentralized board of investors with an incentive to support the structural changes. This increases the stability and security of the whole Giant blockchain network.

JavaScript

JavaScript — the most popular programming language is used in the Giant smart contracts. This seriously increases the number of potential Giant programmers around the world. It also makes the source code of smart contracts easy to comprehend for a wide group of cryptocurrency and information technology enthusiasts.

Code Update

Code Update – allows developers make important changes and/or additions in a smart contract that already exists instead of creating a hundred of iterations of the same code — in the long term this means that the system will not become overrun with obsolete deal data, resulting in better scalability.

Article

Destruction

Smart Contracts Destruction – old Giant smart contracts can be deleted, which prevents a situation where the Giant blockchain system is potentially filled with old data nobody needs anymore. The network scalability becomes even less of a problem if this function is combined with the previously mentioned Code Update feature.

Burning Fees

Burning Fees – to solve the inflationary problem connected to the non-stop network expansion due to the infinite Giant Coin supply, we have introduced the mechanism of burning fees for transactions and smart contract settlement: these commissions are made in Giant Coin (GIC) and burning them will lead to a stable demand for this cryptocurrency.

Self-Governance

Self-Governance – due to the high level of decentralization, the blockchain is able to outlive the original developers as there is no centralized fiscal authority issuing Giant Coin or controlling the smart contracts. Budget decisions are made first with the help of the regular users who hold a vote on Discord, then the voting system of masternodes.
